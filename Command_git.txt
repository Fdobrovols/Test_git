 Создаем пользавательский репазиторий, который скачиваем с сайта, https://git-SCM.com
 устанавливаем клиент
 Для сосдания проекта пишем команду $ git init Для проверки можем посмотреть скрытые папки и увидеть папку.git

 Обязательно настройка проекта сосдания пользавателя
 для того чтобы получить help по команде. Пишем $ git help config "названия команды на пример "config""
 создание пользавателя :
 1. git config --global user.name "Имя пользавателя"
 2. git config --global user.email "Рабочий поштовий ящык" - (это важно для привязки локального репазитория, к глобальному)

 Команда проверок настроек
 git config --list - убеждаемся в правильной записи имени пользавателя и имейла

 git status - выводит информацию о состоянии - используется для проверки команд

 Добавляем файлы под наблюдения gita. Команда git add . - точка значит добавить все файлы проекта или же нужно писать названия файла который ставится под наблюдением (без точки)
 Для добавления файлов в локальный репазиторий используем командку git commit -a (-a: - добавить все файлы в проект)  -m"названия точки сохранения" -m значит что мы создаем точку  востановления(сохранение)
 Для первого комита. Выпосляем команду:
 git commit -a -m" Первая точка сохранения"
 Для последующего добавления изменения выполняем команды
 1. git add .
 2. git commit -m"Названия новой точки сохранения"

 Создаем глобальный репазиторий на сайте https://github.com
 заходим на сайт, и создаем с помощью кнопки new новый репазиторий под этот проект
 Прежде чем создавать глобальный репазиторий нужно обезательно сделать точку ввостановления в локальном репазиторием

 Команда добавления в файле в локальном репазиторий

git remote add origin https://github.com/Fdobrovols/Test_git.git --  Команда сойденяить локальный и глобальный репазиторий
git branch -M main -- Создаем основную ветку Master с названием main
git push -u origin main -- ДОбавляет данные локального репазитория в глобальный

Для просмотра веток пишем команду git branch -v
